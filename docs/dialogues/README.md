# Диалоги с LLM / Агентами

Архив разговоров с языковыми моделями и AI-агентами. Даже неудачные диалоги содержат зёрна идей, которые могут пригодиться позже.

## Зачем хранить диалоги

- **Неудачные подходы** — показывают, где модель ошиблась и почему, это сужает пространство поиска
- **Частичные решения** — модель может пройти 80% пути, а оставшиеся 20% ты доделаешь сам
- **Разные углы зрения** — разные модели подходят к задаче по-разному, их синтез ценнее каждого по отдельности
- **Наводящие вопросы** — иногда модель задаёт хороший вопрос, даже если не даёт правильного ответа

## Как сохранять диалоги

### Вариант 1: Полный экспорт (предпочтительно)

Сохрани диалог целиком как `.md` файл с метаданными в заголовке:
- Файл: `D-{номер}_{тема}_{модель}.md`
- Формат: скопируй `_template.md`

### Вариант 2: Выжимка ключевых моментов

Если диалог длинный — сохрани только важные фрагменты с контекстом.

### Вариант 3: Сырой JSON/текст

Положи экспортированный файл в `raw/` и создай `.md`-обёртку с метаданными и ключевыми выводами.

## Нумерация

`D-001`, `D-002`, ... — последовательная, никогда не переиспользуется.

## Организация

```
dialogues/
├── README.md
├── _template.md
├── raw/                      # Сырые экспорты (JSON, TXT, HTML)
│   └── .gitkeep
└── D-001_example_topic.md    # Обработанные диалоги
```

## Как извлекать пользу

1. После сохранения — перечитай и выдели ключевые идеи
2. Пометь в `## Извлечённые идеи`, какие мысли стоит развить
3. Создай ссылки на гипотезы `[[H-XXX]]` или инсайты `[[I-XXX]]`, если идея оформилась
4. Периодически перечитывай старые диалоги — со временем понимание растёт
