# Мультимодельный анализ

Разные LLM думают по-разному. Эта система позволяет каждой модели анализировать гипотезу в своём рабочем пространстве, а затем сравнивать и синтезировать результаты.

## Параллельный анализ (subagents)

В `.cursor/agents/` настроены analyzer-* subagents. Для запуска анализа одной гипотезы всеми моделями:

```
Запусти в параллель analyzer-claude, analyzer-gpt, analyzer-gemini, analyzer-composer, analyzer-grok, analyzer-kimi, analyzer-codex.

Каждый читает docs/hypotheses/H-XXX_*.md, заполняет docs/reports/_template.md и сохраняет в docs/reports/H-XXX_{model}.md
```

Или попроси агента: **«Запусти мультимодельный анализ H-001»**. Включи Max Mode и Multiple models в Composer.

## Как это работает

```
docs/models/
├── claude/              ← Рабочее пространство Claude
│   ├── profile.md       ← Профиль: сильные стороны, стиль промптинга
│   └── H-001.md         ← Анализ гипотезы H-001 от Claude
├── gpt/                 ← Рабочее пространство GPT
│   ├── profile.md
│   └── H-001.md         ← Анализ той же гипотезы от GPT
├── gemini/              ← Рабочее пространство Gemini
│   └── ...
├── deepseek/            ← Рабочее пространство DeepSeek
│   └── ...
├── o3/                  ← Рабочее пространство o3
│   └── ...
└── _synthesis/          ← Сравнение и синтез
    └── H-001_synthesis.md  ← Синтез по гипотезе H-001
```

## Процесс

### 1. Ставишь задачу модели

Даёшь модели карточку гипотезы `[[H-XXX]]` и просишь проанализировать. Используй `_prompt_template.md` как основу для промпта.

### 2. Модель пишет свой анализ

Результат сохраняется в `models/{model_name}/H-XXX.md` по шаблону `_analysis_template.md`. Модель может создавать любые дополнительные `.md` файлы в своей папке.

### 3. Сравниваешь результаты

Открой анализы от разных моделей бок о бок. Запиши синтез в `_synthesis/H-XXX_synthesis.md`.

### 4. Извлекаешь лучшее

Лучшие идеи из анализов переноси в основную базу знаний:
- Новые подходы → `docs/proofs/`
- Новые наблюдения → `docs/insights/`
- Контрпримеры → `docs/experiments/`

## Зачем несколько моделей

| Преимущество | Описание |
|-------------|----------|
| **Разнообразие** | Каждая модель имеет свои blind spots — вместе они покрывают больше |
| **Верификация** | Если 3 из 4 моделей согласны — это сигнал |
| **Креативность** | Разные модели предлагают разные подходы к доказательству |
| **Поиск ошибок** | Одна модель может найти дыру в рассуждениях другой |

## Как добавить новую модель

1. Создай папку `models/{model_name}/`
2. Скопируй `_model_template/profile.md` и заполни
3. Начни давать задачи — анализы по шаблону `_analysis_template.md`

## Важно

- **Не доверяй слепо** — модели могут уверенно ошибаться, особенно в доказательствах
- **Проверяй вычисления** — прогони через `src/` всё, что можно проверить программно
- **Фиксируй промпты** — записывай, что именно спрашивал, это влияет на качество ответа
