# Inbox — Входная точка для сырых материалов

Сюда кидаешь **что угодно**: голые md-файлы с идеями, переписки с нейросетями, наброски — а **агент** сам всё распознает, оформит в карточки и перенесёт куда надо.

## Куда класть

| Папка | Что класть |
|-------|-----------|
| `raw/` | Любые .md, .txt файлы с идеями, гипотезами, набросками |
| `raw/` | Экспорты переписок с LLM (ChatGPT, Claude, Gemini и т.д.) — .md, .txt, .json |
| `raw/` | Скриншоты в виде текста, заметки «для себя» — всё, что ещё не структурировано |

## Что делает агент при обработке

**Триггер:** Пользователь просит «обработай inbox» или «разбери что в inbox», или явно указывает на файлы в `inbox/raw/`.

**Алгоритм:**

1. **Просмотреть** все файлы в `inbox/raw/` (и подпапках)
2. **Распознать тип содержимого:**
   - **Гипотеза** — утверждение вида «для любого G выполняется X», «верно ли что...», формула с условиями на граф
   - **Инсайт/наблюдение** — паттерн, идея, аналогия, «интересно что...»
   - **Определение/концепция** — формальное определение объекта или свойства
   - **Диалог с LLM** — структура User/Assistant, экспорт чата, переписка
   - **Смешанное** — несколько типов в одном файле
3. **Оформить по шаблонам:**
   - Гипотезы → `hypotheses/H-XXX_{название}.md` (шаблон `hypotheses/_template.md`)
   - Инсайты → `insights/I-XXX_{название}.md` (шаблон `insights/_template.md`)
   - Концепции → `concepts/C-XXX_{название}.md` (шаблон `concepts/_template.md`)
   - Диалоги → `dialogues/D-XXX_{тема}.md` (шаблон `dialogues/_template.md`) + извлечь идеи в I-XXX, H-XXX если есть
4. **Определить домены** для гипотез по `domains/taxonomy.md` (trees, planar, coloring и т.д.)
5. **Проставить связи** `[[H-XXX]]`, `[[I-XXX]]`, `[[C-XXX]]` между карточками
6. **Обновить** `docs/index.md` и индексы в `domains/{domain}/index.md` для затронутых доменов
7. **Создать сводку** в `docs/reports/` по шаблону `reports/_template.md`:
   - Файл: `reports/H-XXX_{model_name}.md` (например `H-001_claude.md`)
   - Самодостаточный документ: формулировка, что обнаружено, что доказано, что не удалось, открытые вопросы, список созданных карточек
   - Подписать именем модели-агента в конце файла
   - Если несколько моделей анализируют одну гипотезу — каждая создаёт свой файл в той же папке
8. **Переместить** обработанные файлы в `inbox/processed/` (сохранить исходник для истории)

## Правила распознавания

### Гипотеза (H-)
- Содержит утверждение о графах, инвариантах, связях вида «для любого...», «верно ли что...»
- Математические формулировки с $...$, LaTeX
- Можно извлечь чёткую формулировку

### Инсайт (I-)
- Наблюдение, паттерн, «кажется что...», «заметил что...»
- Аналогия с другой областью
- Вопрос, который может привести к гипотезе

### Концепция (C-)
- Формальное определение («Граф G называется X, если...»)
- Ссылка на известную теорему/лемму
- Ключевое свойство объекта

### Диалог (D-)
- Чередование сообщений User / Assistant / System
- Явный экспорт чата (ChatGPT, Claude и т.д.)
- Из диалога извлечь: гипотезы → H-XXX, идеи → I-XXX, сохранить сам диалог в D-XXX

## Нумерация при создании

- Посмотреть текущие ID в `docs/index.md` и в папках
- Взять следующий свободный номер: H-001, I-001, C-001, D-001
- Если папка пустая — начинать с 001

## Если не уверен

- Сомнительное — оформить как **инсайт** (I-), потом можно переформулировать в гипотезу
- Непонятный фрагмент — оставить в raw, добавить комментарий в README или создать карточку с пометкой «требует уточнения»
