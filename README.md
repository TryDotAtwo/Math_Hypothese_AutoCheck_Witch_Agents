# Graph Theory Hypothesis Lab

Проект для систематического анализа гипотез в теории графов.

## Философия

Проект построен на трёх принципах:

1. **Zettelkasten для математики** — каждая идея, определение, гипотеза и доказательство живут как атомарная заметка со связями
2. **Вычислительная верификация** — любую гипотезу можно проверить на классах графов программно
3. **Воспроизводимость** — каждый эксперимент задокументирован и может быть повторён

## Быстрый старт

```bash
pip install -r requirements.txt
```

## Рабочий поток: от сырья к карточкам

1. Кидаешь в **`docs/inbox/raw/`** всё подряд: голые md с идеями, экспорты переписок с ChatGPT/Claude/др.
2. Говоришь агенту: **«Обработай inbox»** или **«Разбери что в inbox»**
3. Агент распознает гипотезы, инсайты, диалоги → оформляет по шаблонам → переносит в `hypotheses/`, `insights/`, `dialogues/`
4. Обработанные файлы переезжают в `inbox/processed/`

## Система идентификаторов

Каждая заметка имеет уникальный ID:

| Префикс | Тип | Пример |
|----------|-----|--------|
| `H-` | Гипотеза | `H-001` |
| `C-` | Концепция/определение | `C-012` |
| `P-` | Доказательство | `P-003` |
| `E-` | Эксперимент | `E-007` |
| `I-` | Инсайт/наблюдение | `I-015` |
| `R-` | Ссылка на источник | `R-042` |
| `D-` | Диалог с LLM | `D-003` |

## Три оси навигации

```
       По ID                 По домену              По модели
   (что именно?)          (о каких графах?)      (кто анализировал?)
                    
   H-001 ─────────── trees, coloring ─────── claude/H-001.md
   H-002 ─────────── spectral ──────────── gpt/H-002.md
   H-003 ─────────── planar, extremal ──── _synthesis/H-003.md
```

Одна гипотеза может принадлежать нескольким доменам и быть проанализирована несколькими моделями. Всё связано через `[[ID]]`-ссылки.
